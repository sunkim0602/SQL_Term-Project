## Overview
This project designs and implements a relational database system for managing travel bookings, memberships, and payments. It demonstrates the use of database design principles, relational integrity, and advanced SQL operations.

## Features
- 20+ normalized tables with primary and foreign key relationships  
- Sequences and indexes for efficient data access  
- Stored procedures for handling business logic  
- Triggers for maintaining data consistency  
- Views and queries for analytical reporting  

## Technologies
- SQL Server / T-SQL

## How to Run
1. Run `schema.sql` to create tables, sequences, and indexes.  
2. Run `inserts.sql` to populate sample data.  
3. Execute `procedures.sql` and `triggers.sql` to add logic.  
4. Use `queries.sql` for sample data retrieval and reports.

## Key Learning
This project showcases database architecture design, query optimization, and structured data handling in SQL.
